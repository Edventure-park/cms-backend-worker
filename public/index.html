EVP CMS Backend Worker API

Under Development TypeScript 5.9.2 Cloudflare Workers 4.0

Overview

The EVP CMS Backend Worker is a lightweight, serverless backend service built on Cloudflare Workers — designed to handle dynamic data operations, API requests, and background tasks for the EVP CMS ecosystem. This API powers the entire EVP-CMS platform.

A Modern, Scalable Backend Worker Powering the EVP-CMS

API Features

CI/CD Pipeline Support Automated deployment with GitHub Actions for seamless integration.

Security Features

In Development
  • HTTP Security Headers: Comprehensive security headers including Content Security Policy, Strict-Transport-Security (HSTS), X-Frame-Options, and X-Content-Type-Options to ensure application security.
  • HTTPS Enforcement: HTTPS enforced in production environments with HSTS headers for secure, encrypted connections.
  • CORS Configuration: Strict cross-origin resource sharing policy allowing only trusted origins (localhost:3000, blogs.edventurepark.com).
  • Rate Limiting: Intelligent API rate limiting to prevent abuse and protect sensitive routes like /mail endpoints.
  • Helmet-like Middleware: Custom security middleware adding multiple layers of protection through HTTP headers.
  • Session Management & CSRF Protection: Secure session handling with HttpOnly, SameSite=Strict cookies and anti-CSRF token validation.
  • Logging & Monitoring: Comprehensive logging and real-time monitoring to detect and respond to malicious activity.
  • Content-Type Validation: Strict validation of incoming content types and file sizes to prevent malicious uploads and injection attacks.

API Routes

Below is a comprehensive list of available API endpoints, categorized by their HTTP methods and functionality:

Endpoint Method Description
/mail/send-mail-on-server-1 POST Send an email via Server 1 with specified parameters.
/mail/get-all-sent GET Retrieve a list of all sent emails from the system.
/user/create POST Create a new user account with provided credentials.
/user/get-all GET Retrieve a comprehensive list of all registered users.
/auth/login POST Authenticate user credentials and establish a session.
/auth/logout POST Terminate the current user session and clear credentials.
